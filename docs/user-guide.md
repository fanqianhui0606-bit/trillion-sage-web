# 使用手册：服务流程跟进平台

## 一、环境准备

### 1.1 本地运行
```bash
cd trillion-sage-web
npm run dev
# 打开 http://localhost:3000
```

### 1.2 阿里云部署后访问
```
http://你的阿里云服务器IP/
```

---

## 二、服务流程平台（/tracker）

入口：首页导航栏「服务流程」→ http://localhost:3000/tracker

### 2.1 双入口说明

| 入口 | 用户 | 验证码格式 | 功能 |
|---|---|---|---|
| 家庭客户入口 | 来访家庭 | 6 位字母数字 | 查看流程进度，等待引导员更新 |
| 团队成员登录 | 引导员/学长 | 8 位字母数字 | 创建订单、标记步骤、导出数据 |

### 2.2 引导员操作流程

**首次使用（创建订单）：**

1. 点击导航栏「服务流程」
2. 选择「团队成员登录」
3. 输入 8 位团队成员码（内测阶段任意 8 位字符串均可），输入姓名，进入控制台
4. 输入来访者姓名，选择套餐，点击「生成订单与家庭联合码」
5. 把生成的 **6 位家庭联合码** 告知家长/学生

**日常跟进：**

1. 团队成员码登录 → 自动进入控制台
2. 要跟进某个订单：在控制台→点击「导入订单 JSON」加载家长导出的数据，或让家长重新登录查订单号
3. 或直接让家长登录 → 团队在另一标签页操作 → **跨标签页实时同步**

### 2.3 验证码规则

- **家庭联合码**：6 位字母数字（程序自动生成）
- **团队成员码**：8 位字母数字（内测阶段任意）

## 三、数理素质测验（/quiz）

入口：首页「开始测验」或 http://localhost:3000/quiz

### 3.1 版本区别

| 版本 | 激活码 | 题数 | 维度 |
|---|---|---|---|
| 体验版 | 无 | 17 题 | 8 维 |
| 专业版 | 12 位激活码 | 42 题（14+22+6） | 14 维 + AI 总评 |

### 3.2 激活码
内测阶段：激活码由引导员手动生成或发放。正式运营后需对接支付系统。

### 3.3 AI 总评功能（专业版专有）

专业版完成测验后，结果页会显示「五、测验总评」，由 DeepSeek API 生成。

**激活方法：**
1. 在 GitHub 仓库 Settings → Secrets → Actions 添加环境变量：
   - 名称：`DEEPSEEK_API_KEY`
   - 值：你的 DeepSeek API Key
2. 部署到阿里云后，在服务器 `.env.production` 文件中添加：
   ```env
   DEEPSEEK_API_KEY=sk-your-key-here
   ```
3. 重启 PM2：`pm2 restart trillionsage-web`

**未配置时的行为：** 自动降级为本地规则模板（无需 API Key）。

## 四、敏感信息清单

| 内容 | 存放位置 | 注意事项 |
|---|---|---|
| DeepSeek API Key | 服务器 `.env.production` / GitHub Secrets | **禁止提交到 GitHub 代码库** |
| 团队成员码 | 内存/文档 | 内测阶段可随意，正式运营存数据库 |
| 家庭联合码 | localStorage | 临时存储，正式运营迁到服务端 |
| 来访者信息 | localStorage | 临时存储，正式运营需加密存数据库 |

### 4.1 本地开发环境变量
```bash
# 在 trillion-sage-web 根目录创建 .env.local
DEEPSEEK_API_KEY=sk-your-key-here
```
`.env.local` 已加入 `.gitignore`，不会 push。

## 五、家庭使用流程

1. 打开服务流程页面（引导员发链接）
2. 选择「家庭客户入口」
3. 输入 6 位家庭联合码 + 姓名
4. 查看当前流程进度（步骤自动实时更新，无需刷新）
5. 等待引导员标记完成各步骤
